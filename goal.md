| Automatically saves conversation after prompt + agent output? | **Partially.** `startSession` saves the user prompt and context pack. `finishSession` can create a learning draft from prompt, selected context, decisions, and the finish `summary`. It **does not automatically capture the full raw conversation or full agent output** unless the agent submits it as a summary, note, reflection draft, or error log. See [agent-session/service.ts](/home/nash/tuberosa/src/agent-session/service.ts:37) and [agent-session/service.ts](/home/nash/tuberosa/src/agent-session/service.ts:344). |
| Saves multiple knowledge formats? | **Yes, for text/structured knowledge.** Supported item types are `spec`, `workflow`, `memory`, `bugfix`, `code_ref`, `rule`, `wiki`, and `conversation`. Files can be ingested as documents or atomized markdown sections. See [types.ts](/home/nash/tuberosa/src/types.ts:1) and [ingest/service.ts](/home/nash/tuberosa/src/ingest/service.ts:61). |
| Auto-approves pending knowledge? | **Not generally.** Manual reflection drafts stay reviewable until approved. But session auto-learning can auto-approve a strong candidate when strict gates pass: completed session, compliant context decision, ready context pack, no negative feedback, no duplicate, grounded references, concrete labels, enough content. Otherwise it becomes `needs_changes`. See [agent-session/service.ts](/home/nash/tuberosa/src/agent-session/service.ts:236) and [agent-session/service.ts](/home/nash/tuberosa/src/agent-session/service.ts:371). |
| Is matching noise minimized? | **Improved, but not guaranteed minimal.** Tuberosa has context-fit thresholds, direct/adjacent evidence categories, feedback penalties, stale/rejected handling, and domain mismatch demotion. But vague prompts can still retrieve adjacent/noisy context. The new completeness benchmark measures `noiseRate`, but that proves benchmark cases, not all real use. See [context-fit.ts](/home/nash/tuberosa/src/retrieval/context-fit.ts:5) and [context-pack.ts](/home/nash/tuberosa/src/retrieval/context-pack.ts:198). |
| Friendly for non-tech users who just prompt? | **Not yet fully.** Tuberosa supports this direction, but today it is mainly an MCP/HTTP context broker. A user can prompt normally, but the **agent must call** `tuberosa_start_session`, record a context decision, and finish the session. Tuberosa does not yet act as a full autonomous orchestrator that takes a plain user prompt and triggers all work by itself. See [docs/SETUP_AND_USAGE.md](/home/nash/tuberosa/docs/SETUP_AND_USAGE.md:748). |

My read: your goal is valid, but two areas need product decisions.

1. **Memory trust policy:** If your goal is “never auto-approve pending knowledge,” change the default from `learningMode: "auto"` to `draft_only`, or add a project-level setting. Current behavior intentionally allows strict auto-approval, and the audit calls that a policy tension: [docs/FLOW_INTENT_AUDIT.md](/home/nash/tuberosa/docs/FLOW_INTENT_AUDIT.md:63).

2. **Non-tech UX:** Tuberosa has the backend pieces, but needs a simple front door: “type prompt → Tuberosa starts session → retrieves context → asks/acts via agent → records result → drafts memory.” Today that workflow exists as protocol, not as a polished user experience.
